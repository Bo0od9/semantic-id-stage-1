# Стратегия проведения ablation-экспериментов

Этот документ фиксирует стратегию проведения дорогих по числу запусков разделов из плана `experiment_plan_revised.md`: раздел 2.2 (подбор глубины D и размера кодовой книги K для RQ-VAE) и раздел 3.3 (варьирование ёмкости кодов в экспериментах 3.1 и 3.2). Остальные разделы плана выполняются без изменений.

Цель документа — один раз зафиксировать компромисс между объёмом вычислений и содержательностью ablation, чтобы в процессе экспериментов не возвращаться к этому обсуждению.

## Контекст

Исходный план предполагал:

- В разделе 2.2 — 7 уникальных конфигураций RQ-VAE × 3 запуска с разными случайными начальными значениями = 21 запуск.
- В разделе 3.3 — те же 7 конфигураций × 2 схемы (3.1 и 3.2) × 3 запуска = 42 запуска.

Итого 63 полноценных обучения только ради ablation-кривых, ещё до главных экспериментов. Такой объём на MacBook M4 Pro занимает несколько недель.

Стратегия ниже сокращает объём ablation примерно на 90 % без потери содержательности — главные эксперименты (3.1, 3.2, 3.5, 3.6, 3.8) проводятся в полном объёме и не затрагиваются.

## Три принципа сокращения

### 1. Меньше конфигураций

Вместо 7 — 5 уникальных конфигураций:

- По оси глубины: `D ∈ {2, 4, 8}` при `K = 256` — три точки (минимум, рабочая, крайнее значение). Промежуточные `D = 6` и `D = 1` не запускаются: при монотонной зависимости качества от глубины они не добавляют информации.
- По оси размера кодовой книги: `K ∈ {64, 256, 1024}` при `D = 4` — три точки. Конфигурация `K = 32` не запускается, так как она нужна только для 20-битного варианта контроля с k-means (раздел 3.5), а 3.5 проводится в 16-битной версии (`D = 2, K = 256`).
- Рабочая конфигурация `D = 4, K = 256` — общая точка обеих осей.

Итого 5 уникальных конфигураций: `(2, 256)`, `(4, 64)`, `(4, 256)`, `(4, 1024)`, `(8, 256)`.

### 2. Меньше случайных начальных значений на ablation-точки

Стандартное правило плана — три запуска с начальными значениями 42, 43, 44 для получения среднего и стандартного отклонения. Это оправдано для главных экспериментов, но избыточно для ablation, где цель — увидеть форму кривой, а не точные числа.

- **Рабочая конфигурация `D = 4, K = 256`:** все три запуска (42, 43, 44). Цифры по ней идут в итоговую таблицу диплома.
- **Остальные четыре ablation-конфигурации:** один запуск на seed = 42.

Если точка на кривой выглядит аномально (резко выбивается из монотонного тренда, или стандартное отклонение на соседних точках много меньше величины отклонения), дозапускается ещё один-два запуска с seed = 43, 44 для этой конкретной конфигурации. Это делается выборочно, по факту.

### 3. Ablation — на 10 % подмножестве, главные эксперименты — на полном 50M

Разделы 2.2 и 3.3 проводятся на 10 % подмножестве данных (примерно 5 миллионов взаимодействий). Это подмножество уже подготовлено на стадии предобработки (`train_subsample_10pct.parquet` и аналоги).

Запуски на 10 % подмножестве занимают в 5–10 раз меньше времени, чем на полных 50 М, и дают ту же форму кривой. Оптимальная конфигурация, выбранная на 10 %, перед главными экспериментами переобучается на полных 50 М.

Главные эксперименты (3.1, 3.2, 3.5, 3.6, 3.8 в основной конфигурации) — на полном 50 М с тремя запусками, как в плане.

## Конкретная сетка запусков

### Раздел 2.2 (обучение RQ-VAE)

На 10 % подмножестве:

| Конфигурация | Число запусков | Начальные значения |
|---|---|---|
| D = 2, K = 256 | 1 | 42 |
| D = 4, K = 64 | 1 | 42 |
| D = 4, K = 256 (рабочая) | 3 | 42, 43, 44 |
| D = 4, K = 1024 | 1 | 42 |
| D = 8, K = 256 | 1 | 42 |

Итого 7 запусков RQ-VAE на 10 %.

Дополнительно на полном 50 М:

| Конфигурация | Число запусков | Начальные значения |
|---|---|---|
| D = 4, K = 256 (рабочая, для главных экспериментов) | 3 | 42, 43, 44 |

Итого 10 запусков RQ-VAE вместо 21 по исходному плану.

### Раздел 3.3 (варьирование D и K в 3.1 и 3.2)

На 10 % подмножестве, для каждой из 5 конфигураций — по две схемы (3.1 с обучаемыми таблицами и 3.2 с замороженными векторами):

| Конфигурация | Схема 3.1 | Схема 3.2 |
|---|---|---|
| D = 2, K = 256 | 1 запуск (seed = 42) | 1 запуск (seed = 42) |
| D = 4, K = 64 | 1 запуск | 1 запуск |
| D = 4, K = 256 (рабочая) | 3 запуска (42, 43, 44) | 3 запуска (42, 43, 44) |
| D = 4, K = 1024 | 1 запуск | 1 запуск |
| D = 8, K = 256 | 1 запуск | 1 запуск |

Итого `4 × 2 × 1 + 1 × 2 × 3 = 14` запусков вместо 42 по исходному плану.

### Общая экономия

- Исходный план ablation: 63 запуска.
- По этой стратегии: 10 + 14 = 24 запуска.
- Сокращение примерно в 2.5 раза по числу запусков, и дополнительно в 5–10 раз по времени каждого запуска за счёт перехода на 10 % подмножество (кроме одной рабочей конфигурации на полном 50 М).

## Порядок выполнения

1. **Сначала SASRec-ID и SASRec-Content.** Три запуска каждой на полном 50 М. Это предварительный этап, необходимый для всех остальных экспериментов. После обучения для каждого пользователя сохраняются `z_u^{ID}` и `z_u^{содерж.}` — они кэшируются и переиспользуются, чтобы не прогонять SASRec повторно.

2. **PCA на полном `z_u^{содерж.}`.** Быстрый этап (линейная алгебра), выполняется один раз.

3. **Раздел 2.1 и 2.2 — RQ-VAE на 10 % подмножестве.** Семь запусков по таблице выше. Для каждой конфигурации записывается ошибка восстановления и доля используемых кодов.

4. **Дообучение рабочей конфигурации RQ-VAE на полном 50 М.** Три запуска `D = 4, K = 256` — эти кодовые книги и коды пользователей используются во всех главных экспериментах.

5. **Проверочные эксперименты 2.3, 2.4, 2.5 — на полном 50 М.** Не требуют обучения, только применение готовых моделей и поиск по таблицам.

6. **Раздел 3.3 — варьирование в схемах 3.1 и 3.2, на 10 % подмножестве.** Четырнадцать запусков по таблице выше. Получаются две параллельные кривые — с обучаемыми таблицами и с замороженными векторами.

7. **Главные эксперименты 3.1 и 3.2 — на полном 50 М.** По три запуска каждой в рабочей конфигурации `D = 4, K = 256`. Это основные цифры для таблицы результатов диплома.

8. **Контроль 3.5 — k-means.** Один запуск k-means на полном `z_u^{содерж.}`, затем три запуска 3.2-варианта и три запуска k-means-варианта с полученной замороженной таблицей.

9. **Эксперименты 3.6 (подмена кода) и 3.8 (замена непрерывного вектора).** Только применение моделей, обучение не требуется.

## Правила эскалации при аномальных результатах

**Случай 1. Ablation-точка с одним запуском резко выбивается из общего тренда.**

Пример: кривая качества монотонно растёт от `D = 2` к `D = 4`, а в точке `D = 8` резкое падение, которого не ожидалось. Действие: дозапустить ту же точку с seed = 43 и seed = 44. Если падение подтверждается на всех трёх запусках, это содержательный результат (избыточная ёмкость вредит). Если нет — первая точка была случайным выбросом.

**Случай 2. Рабочая конфигурация на 10 % и на 50 М ведёт себя сильно по-разному.**

Пример: на 10 % `D = 4, K = 256` значимо лучше `D = 2, K = 256`, а на 50 М это различие пропадает. Действие: проверить, обоснована ли рабочая конфигурация. Если выбор конфигурации по 10 % оказался ненадёжным, перепроверить ablation на 50 М хотя бы для двух соседних конфигураций.

**Случай 3. Доля используемых кодов в RQ-VAE меньше 50 % на какой-либо конфигурации.**

Это признак вырождения кодовой книги. Переобучить с более агрессивным сбросом неиспользуемых кодов и увеличенным числом итераций. Конфигурация включается в результаты только после того, как доля используемых кодов превысит 50 %.

**Случай 4. На 10 % подмножестве ни одна из конфигураций не показывает улучшения над SASRec-ID baseline в 3.1.**

Ни одна из 5 конфигураций не даёт значимого прироста. Действие: перед переходом к полному 50 М проверить ошибку восстановления RQ-VAE и результат 2.3 (точное совпадение кода). Если они разумные, возможно, что подмножество 10 % слишком маленькое для того, чтобы коды проявили эффект. В этом случае прогоняется одна конфигурация (`D = 4, K = 256`) на полном 50 М для проверки. Если и на 50 М эффекта нет — это содержательный отрицательный результат, и эксперимент 3.3 на 50 М проводить не нужно.

## Что этот документ не меняет

- Список главных экспериментов из плана (3.1, 3.2, 3.5, 3.6, 3.8) — без изменений.
- Метрики, функции потерь, протокол временнóго разделения — без изменений.
- Структуру проверочных экспериментов (2.3, 2.4, 2.5) — без изменений.
- Стандарт «три запуска с начальными значениями 42, 43, 44» для главных экспериментов — без изменений.

Если по ходу экспериментов выяснится, что эта стратегия приводит к потере содержательных деталей (например, кривая по D окажется немонотонной и понадобится точка `D = 6`), документ обновляется точечно: добавляется одна-две дополнительных конфигурации, остальное остаётся как есть.
